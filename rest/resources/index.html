Announcements

Announcements are general news about the system. They are shown in the /agent interface until dismissed. Dismissing an announcement should mark it as read.

GET /announcements

Response:

{
  "announcements": [
    {
      "id": 1,
      "title": "Example announcement",
      "body": "<p>Some text that might contain HTML</p>",
      "image_url": "http://example.com/image.png",
      "button_title": "Read more",
      "button_url": "http://firmafon.dk/read_more"
    }
  ]
}

POST /announcements/:id/mark_as_read

Parameters:

id

Required: ID of the announcement

Response:

{
  "announcement": {
    "id": 1,
    "title": "Example announcement",
    "body": "<p>Some text that might contain HTML</p>",
    "image_url": "http://example.com/image.png",
    "button_title": "Read more",
    "button_url": "http://firmafon.dk/read_more"
  }
}

Calls

Oh Calls, where do we begin?

If all you’re trying to do is get some historical data then there’s not much to it.

If you want to understand how it’s all connected, read the detailed Calls guide.

GET /calls

Parameters:

limit

Specify how many you want. Default: 30, Maximum: 100.

endpoint

Filter calls by endpoint. Example: Reception#123.

number

Filter calls by a number. Either to or from. Example: 4512345678.

started_at_lt_or_eq

Page the calls by using the started_at attribute of the last of the already fetched calls. Example: 2014-04-04 14:07:41 +0200.

CallNodes

A CallNode is a … OH GOD WHERE DO I START

If you want to understand how it’s all connected, read the detailed Calls guide.

GET /call_nodes/live

Get all current live nodes for the authorized user.

Response:

{
  "call_nodes": [
    {
      "node_uuid": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
      "call_uuid": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
      "endpoint": "Employee#1",
      "started_at": "2014-03-26T13:03:56Z",
      "answered_at": "2014-03-26T13:03:59Z",
      "ended_at": null,
      "state": "answered",
      "node_type": null
    }
  ]
}

POST /call_nodes/:node_uuid/cancel_transfer

Cancel an active transfer. This is also what you want to do if you want to take back the call.

Parameters:

node_uuid

Required. UUID of the node.

Returns status code 200.

POST /call_nodes/:node_uuid/node_uuid/hangup

Hang up the given CallNode.

Parameters:

node_uuid

Required. UUID of the node.

Returns status code 200.

POST /call_nodes/:node_uuid/transfer

Start a transfer of the given call node to the given endpoint.

Parameters:

node_uuid

Required. UUID of the node.

endpoint

Required. What to transfer to. Example: Reception#1.

type

Required. Type of transfer.
Possible values: [attended|direct].

GET /call_nodes/:node_uuid

Get info about a given CallNode.

Parameters:

node_uuid

Required. UUID of the node.

Response:

{
  "call_node": {
    "node_uuid": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
    "call_uuid": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
    "endpoint": "Employee#1",
    "started_at": "2014-03-26T13:03:56Z",
    "answered_at": "2014-03-26T13:03:59Z",
    "ended_at": null,
    "state": "answered",
    "node_type": null
  }
}

Contacts

Contacts are additional info about a given phone number. They are either provided by Firmafon’s internal lookups or by the employee or their colleagues ie. through an Integration.

GET /contacts

Parameters:

number

Required. Example: 4512345678

Response:

{
  "contact": {
    "id": 1,
    "number": "4512345678",
    "name": "Karsten Kontakt",
    "email": "karstenkontakt@example.com"
  }
}

POST /contacts

Create a contact. These map to the fields to_contact and from_contact in Calls.

Parameters:

contact[number]

4512345678

contact[name]

Karsten Kontaktbegreb

contact[email]

karsten@example.com

Response:

{
  "contact": {
    "id": 1,
    "number": "4512345678",
    "name": "Karsten Kontakt",
    "email": "karstenkontakt@example.com"
  }
}

POST /employees/:id/message

Send an SMS to the given employee. **NB: These are cheap but not free - Look for “Web SMS”.

The sender will be shown as either the current employee’s number or name.

Not available for companies in trial.

Parameters:

id

Required: ID of the employee

message[body]

Required: The contents of your message.

Response:

{
  "sent": true
}
